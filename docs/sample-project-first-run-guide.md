# 샘플 프로젝트 처음 시작 가이드

이 문서는 현재 PC에서 샘플 프로젝트를 처음 사용할 때 필요한 메뉴와 버튼만 순서대로 안내합니다. 기능 설명을 모두 읽기보다, 먼저 Project Chat까지 준비하고 싶은 경우 이 문서부터 따라가면 됩니다.

## 지금 상태에서는 바로 채팅할 수 있습니다

2026년 7월 22일 현재 `Sample Shop Demo (#1)`은 모든 준비가 끝난 상태입니다.

| 항목 | 현재 상태 |
|---|---|
| Git | commit 48건, 변경 파일 106건 |
| 프로그램/개발계획 | 8건 |
| Mapping | commit 48/48 완료, 관계 38건 |
| Risk Analysis | 32건 |
| 현재 소스 검색 준비 | 79/79 |
| Embedding | `text-embedding-nomic-embed-text-v2-moe`, 768차원 |
| Knowledge Graph | 213 nodes, 590 edges |
| Project Chat | 사용 가능, 검증한 저장 대화 `#32` 있음 |

따라서 지금은 `전체 수집`, `Mapping`, `전체 소스 다시 읽기`, `전체 재동기화`를 다시 실행할 필요가 없습니다.

### 지금 바로 질문하는 순서

1. 브라우저에서 `http://127.0.0.1:8501/?project_id=1`을 엽니다.
2. 왼쪽 위 `현재 프로젝트`가 `Sample Shop Demo (1)`인지 확인합니다.
3. 왼쪽 메뉴에서 `분석 실행`을 펼칩니다.
4. `Project Chat`을 누릅니다.
5. 화면 위 상태가 아래와 같은지 확인합니다.
   - `소스 근거`: 79
   - `검색 준비`: 79/79
   - `코드 반영 상태`: 최신
   - `추가 준비 필요`: 0
6. `TOP K`는 8로 두고 `커밋 이력도 참고에 포함`은 끈 상태로 둡니다.
7. 기존 결과를 보려면 `저장된 대화`에서 `#32`를 선택합니다.
8. 새 질문을 하려면 `새 대화 시작`을 누른 뒤 화면 아래 `프로젝트에 대해 질문하세요.` 입력란에 질문을 쓰고 Enter를 누릅니다.

처음 질문으로는 아래 문장을 권장합니다.

```text
결제금액 검증은 어디에서 수행되나요?
```

관계 근거까지 보고 싶으면 아래 질문을 사용합니다.

```text
PaymentController.java, PaymentService.java, OrderStatusService.java, OrderStatusMapper.java 기준으로 결제 금액 검증 조건과 승인 후 주문이 PAID가 되는 직접 호출 흐름을 설명해줘. 직접 호출과 간접 연결을 구분해줘.
```

답변 아래에 provider/model, 검증 상태, 현재 소스 근거와 그래프 관계 근거가 표시되면 정상입니다. `fallback=True`가 표시되더라도 현재 소스 검증을 거친 `deterministic_repair`라면 모델 호출 실패나 mock 답변을 뜻하지 않습니다.

## 무엇이 채팅에 꼭 필요한가

옵션을 모두 준비해야 채팅할 수 있는 것은 아닙니다.

| 준비 항목 | 일반 소스 질문 | 프로그램·커밋 관계 질문 | Dashboard/Risk까지 사용 |
|---|---:|---:|---:|
| 프로젝트와 Git 경로 | 필수 | 필수 | 필수 |
| 현재 소스 인덱스와 embedding | 필수 | 필수 | 권장 |
| 프로그램 목록과 개발계획 | 선택 | 필수 | 필수 |
| Mapping | 선택 | 필수 | 필수 |
| Risk Analysis | 불필요 | 불필요 | 필수 |
| Knowledge Graph | 선택 | 필수 | 권장 |
| 표준용어/표준단어 | 권장 | 권장 | 선택 |

현재 샘플 프로젝트는 위 항목이 모두 준비되어 있습니다.

## PC를 다시 켠 뒤 실행하는 순서

현재 설치와 모델 설정은 이미 끝나 있으므로 다음은 PC 재부팅 뒤 서비스만 다시 시작하는 절차입니다.

1. Docker Desktop을 실행합니다.
2. LM Studio를 실행합니다.
3. PowerShell에서 프로젝트 폴더 `C:\dev\ai-commit-advisor`로 이동합니다.
4. LM Studio server와 모델이 실행되지 않은 경우 아래 명령을 실행합니다.

```powershell
lms server start --port 12345
lms load qwen2.5-coder-7b-instruct --identifier qwen2.5-coder-7b-instruct --context-length 8192 --gpu max --parallel 1 --yes
lms load text-embedding-nomic-embed-text-v2-moe --identifier text-embedding-nomic-embed-text-v2-moe --context-length 512 --gpu max --yes
docker compose up -d
```

5. 준비 상태를 확인합니다.

```powershell
.\scripts\demo_preflight.ps1
```

마지막 줄이 `FAIL=0, WARN=0`이면 브라우저에서 `http://127.0.0.1:8501/?project_id=1`을 열면 됩니다. 처음 설치하는 PC라면 이 문서 대신 [설치와 운영](setup-and-operations.md)의 설치 절차부터 진행합니다.

## 빈 DB에서 처음부터 다시 만드는 버튼 순서

아래 절차는 프로젝트 데이터가 하나도 없을 때 사용합니다. 현재 `Sample Shop Demo`는 이미 완성되어 있으므로 지금 다시 실행할 필요는 없습니다.

### 1. 프로젝트 등록

왼쪽 메뉴 `프로젝트 설정 > 프로젝트/Git 설정`으로 이동합니다.

1. `프로젝트 선택`에서 `새 프로젝트`를 고릅니다.
2. `저장소 등록 방식`에서 `서버에 이미 있는 저장소 사용`을 고릅니다.
3. 다음 값을 입력합니다.

| 입력란 | 값 |
|---|---|
| 프로젝트명 | `Sample Shop Demo` |
| 앱 서버 Git 저장소 경로 | `C:\dev\ai-advisor-sample-shop` |
| Git remote URL | 비워 둠 |
| Git branch | `main` 그대로 둠 |
| 설명 | `AI Commit Advisor 샘플 프로젝트` |

4. `프로젝트 저장`을 누릅니다.
5. 왼쪽 위 `현재 프로젝트`가 `Sample Shop Demo`로 바뀌었는지 확인합니다.

`서버 저장소 clone/fetch`는 누르지 않습니다. 샘플 Git 저장소가 이미 `C:\dev\ai-advisor-sample-shop`에 있기 때문입니다.

### 2. Git 전체 수집

왼쪽 메뉴 `프로젝트 설정 > Git 동기화`로 이동합니다.

1. `전체 수집`을 누릅니다.
2. 작업이 끝날 때까지 기다립니다.
3. commit 48건과 변경 파일 106건이 저장됐는지 확인합니다.

첫 실행에서는 `증분 동기화`를 누르지 않습니다. 이 버튼은 나중에 새 commit만 추가로 가져올 때 사용합니다.

### 3. 개발자 목록 업로드

왼쪽 메뉴 `산출물 관리 > 개발자 목록`으로 이동합니다.

1. `Excel 업로드` 탭을 누릅니다.
2. `샘플 개발자 Excel 다운로드`를 누릅니다.
3. 내려받은 `sample_developers.xlsx`를 바로 아래 업로더에서 선택합니다.
4. 검증 결과가 `오류 0`, `저장 가능 6`인지 확인합니다.
5. `검증 통과 행 저장`을 누릅니다.

### 4. 프로그램 목록 업로드

왼쪽 메뉴 `산출물 관리 > 프로그램 목록`으로 이동합니다.

1. `Excel 업로드` 탭을 누릅니다.
2. `샘플 프로그램 Excel 다운로드`를 누릅니다.
3. 내려받은 `sample_programs.xlsx`를 바로 아래 업로더에서 선택합니다.
4. 화면의 컬럼 연결은 자동 선택된 값을 그대로 둡니다.
5. 검증 결과가 `오류 0`, `저장 가능 8`인지 확인합니다.
6. `검증 통과 행 저장`을 누릅니다.

### 5. 개발계획 업로드

왼쪽 메뉴 `산출물 관리 > 개발계획`으로 이동합니다.

1. `Excel 업로드` 탭을 누릅니다.
2. 화면의 안내대로 개발자 목록과 프로그램 목록이 먼저 저장되었는지 확인합니다.
3. `샘플 개발계획 Excel 다운로드`를 누릅니다.
4. 내려받은 `sample_development_plan.xlsx`를 바로 아래 업로더에서 선택합니다.
5. 검증 결과가 `오류 0`, `저장 가능 8`인지 확인합니다.
6. `검증 통과 행 저장`을 누릅니다.

### 6. 표준용어/표준단어 업로드

왼쪽 메뉴 `산출물 관리 > 표준용어/표준단어`로 이동합니다.

1. `Excel 업로드` 탭을 누릅니다.
2. `샘플 표준용어/표준단어 Excel 다운로드`를 누릅니다.
3. 내려받은 `sample_standard_terms.xlsx`를 바로 아래 업로더에서 선택합니다.
4. 검증 결과가 `오류 0`, `저장 가능 14`인지 확인합니다.
5. `검증 통과 행 저장`을 누릅니다.

### 7. Mapping 실행

왼쪽 메뉴 `분석 실행 > Mapping`으로 이동합니다.

1. `분석 모드`는 기본값 `커밋 기준 분석`으로 둡니다.
2. `커밋당 후보 프로그램 TOP N`은 기본값 10으로 둡니다.
3. `실패 커밋도 다시 시도`는 체크된 상태로 둡니다.
4. `미완료 커밋 전체 분석`을 누릅니다.
5. 진행률이 48/48이 될 때까지 기다립니다.

완료 기준은 `완료 커밋 48`, `실패 커밋 0`입니다. program-commit 관계 행 수는 모델 판단에 따라 조금 달라질 수 있으며 현재 저장 결과는 38건입니다.

첫 실행에서는 `프로그램 기준 분석`, `선택한 커밋 분석`, `완료 커밋도 재분석`을 사용하지 않습니다.

### 8. Risk Analysis 실행

왼쪽 메뉴 `분석 실행 > Risk Analysis`로 이동합니다.

1. `리스크 분석 실행`을 누릅니다.
2. 현재 샘플 기준 HIGH 8, MEDIUM 15, LOW 9로 총 32건이 표시되는지 확인합니다.

### 9. 현재 소스와 embedding 준비

왼쪽 메뉴 `분석 실행 > RAG 검색`으로 이동합니다.

1. `한 번에 준비` 탭을 선택합니다.
2. `전체 소스 다시 읽은 뒤 검색 준비도 실행`을 체크합니다.
3. 새로 나타나는 `검색 준비 최대 처리 수`를 100으로 바꿉니다.
4. `전체 소스 다시 읽기`를 누릅니다.
5. 작업이 끝난 뒤 아래 상태를 확인합니다.
   - `소스 근거`: 79
   - `검색 준비`: 79/79
   - `코드 반영 상태`: 최신
   - `추가 준비 필요`: 0

`근거 조각 크기`, `겹치는 글자 수`, `준비할 근거 종류`는 바꾸지 않습니다.

검색 준비가 50/79처럼 일부만 끝났다면 `검색 준비` 탭으로 이동합니다. `검색 준비 대상`에서 `현재 소스 파일`만 남기고, `이번 실행에서 처리할 최대 근거 수`를 100으로 바꾼 뒤 `검색 준비 실행`을 누릅니다.

### 10. Knowledge Graph 준비

왼쪽 메뉴 `분석 결과 > Knowledge Graph`로 이동합니다.

1. Neo4j가 `연결`로 표시되는지 확인합니다.
2. 첫 실행에서는 `전체 재동기화`를 누릅니다.
3. 현재 저장 결과 기준 213 nodes, 590 edges인지 확인합니다.

`최신 변경분만 Neo4j 반영`은 이후 새 commit을 동기화했을 때 사용합니다.

### 11. Project Chat 사용

왼쪽 메뉴 `분석 실행 > Project Chat`으로 이동합니다.

1. `검색 준비 79/79`, `코드 반영 상태 최신`, `추가 준비 필요 0`을 확인합니다.
2. `TOP K`는 8로 둡니다.
3. 처음에는 `커밋 이력도 참고에 포함`을 체크하지 않습니다.
4. 화면 아래 질문 입력란에 질문을 쓰고 Enter를 누릅니다.

이 단계까지 완료하면 소스 기반 Project Chat을 사용할 수 있습니다. `AI Code Review`, `PL Briefing`, Dashboard 확인은 채팅 사용에 필수 단계가 아닙니다.

## 새 commit이 생긴 뒤 다시 누를 버튼

처음부터 다시 만들 필요는 없습니다. 다음 순서만 실행합니다.

1. `프로젝트 설정 > Git 동기화 > 증분 동기화`
2. `분석 실행 > RAG 검색 > 한 번에 준비 > 최신 변경분 반영`
3. `분석 실행 > RAG 검색 > 검색 준비 > 검색 준비 실행`
4. `분석 실행 > Mapping > 미완료 커밋 전체 분석`
5. `분석 실행 > Risk Analysis > 리스크 분석 실행`
6. `분석 결과 > Knowledge Graph > 최신 변경분만 Neo4j 반영`

그다음 `Project Chat`에서 새 질문을 하면 최신 Git HEAD와 현재 소스를 기준으로 답합니다.

## 누르지 말아야 할 버튼

- `프로젝트 삭제`: 프로젝트와 업로드·분석 결과를 모두 지울 때만 사용합니다.
- `분석 데이터 초기화`: 같은 프로젝트의 Git 수집과 분석을 처음부터 다시 만들 때만 사용합니다.
- `완료 커밋도 재분석`: 모델 결과를 의도적으로 새로 만들 때만 사용합니다.
- `프로그램 기준 분석`: 기본 샘플 흐름에서는 사용하지 않습니다.
- `전체 소스 다시 읽기`: 첫 준비 또는 브랜치가 크게 바뀐 경우에만 사용합니다.
- `전체 재동기화`: 첫 graph 준비 또는 관계가 크게 어긋난 경우에만 사용합니다.

## 문제가 생겼을 때

먼저 아래 명령을 실행합니다.

```powershell
.\scripts\demo_preflight.ps1
```

| 실패 위치 | 먼저 확인할 것 |
|---|---|
| Chat model | LM Studio에 `qwen2.5-coder-7b-instruct`가 로드됐는지 확인 |
| Embedding model | `text-embedding-nomic-embed-text-v2-moe`가 로드됐는지 확인 |
| PostgreSQL/Neo4j/App | Docker Desktop과 `docker compose ps` 확인 |
| 검색 준비 부족 | `RAG 검색 > 검색 준비`에서 현재 소스 파일 대상으로 나머지 실행 |
| 코드 반영 상태가 오래됨 | Git 동기화 후 `최신 변경분 반영` 실행 |
| Graph가 오래됨 | `최신 변경분만 Neo4j 반영` 실행 |

더 자세한 기능 설명은 [AI Commit Advisor 사용 가이드](demo-user-guide.md), 설치와 모델 설정은 [설치와 운영](setup-and-operations.md)을 참고합니다.
